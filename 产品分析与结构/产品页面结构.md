# AI智能搜题

`AI智能搜题` 包含 `前端` 与 `后端` 两部分，用于支持 `搜一搜（搜题板块）`、`错题本（错题板块）`、`思考模式`、`举一反三` 与 `错题本分析` 等功能流程。

## 通用交互与约束

- `搜一搜（搜题板块）`：用户上传照片后，`搜索栏` 从页面中部下降到页面底部，`主页面` 开启 `对话框模式`。
- `左侧边栏`：一般情况下不完全展示；完全展示时显示 `过往搜题`。
- `过往搜题`：根据 `题目类型` 自动分类收纳；固定类型包括 `数学`、`语文`、`英语`。
- `错题本（错题板块）`：按照 `科目` 对错题进行分类，并显示每个科目的 `错题数量`。
- `举一反三`：页面下方一开始遮挡 `答案`，由用户主动选择是否展示答案；展示答案之后，主动询问用户是否开启 `思考模式` 进行讲解。
- `思考模式`：系统在题目解析后询问用户是否进入 `思考模式`，且需要推荐 `思考模式`。
- `直接生成答案`：当用户拒绝进入 `思考模式` 时，系统直接生成答案，并在界面下方提供开关，使用户可随时进入 `思考模式`。
- `思考模式循环`：当用户进入 `思考模式` 后，系统主动询问用户哪里不会，并根据用户回答结合解题步骤进行针对性回答；若用户无法独立解答，则继续询问是否是下一个知识点没懂或不会应用，进入循环，直到用户表示完全理解题目。
- `错题本载入`：系统主动询问用户是否将本题载入 `错题本`；拒绝则不记录本题目与本次对话；同意则将本题目记录进 `错题本`，并总结此次对话，生成用户的 `知识薄弱点`。

## 【前端】产品页面功能描述及交互描述

### 说明

- `前端` 包含 `搜一搜（搜题板块）` 与 `错题本（错题板块）`。
- `搜一搜（搜题板块）` 包含 `右侧主页面` 与 `左侧边栏`。
- `错题本（错题板块）` 包含 `主页面` 与 `右上角` 的 `举一反三` 功能入口。

### 搜一搜（搜题板块）

- 页面区域：
  - `右侧主页面`
  - `左侧边栏`

### 右侧主页面

- 页面用途：
  - 展示主要搜索界面。
- 交互行为：
  - 上传照片后，将 `搜索栏` 从页面中部下降到页面底部。
  - 上传照片后，`主页面` 开启 `对话框模式`。

### 左侧边栏

- 默认展示：
  - 一般情况下不完全展示。
- 完全展示：
  - 完全展示时显示 `过往搜题`。
- 分类规则：
  - `过往搜题` 会根据 `题目类型` 自动分类收纳。
  - 固定类型包括 `数学`、`语文`、`英语`。

### 错题本（错题板块）

- 页面区域：
  - `主页面`
  - `右上角`

### 主页面

- 分类规则：
  - 按照 `科目` 对错题进行分类。
- 分类类别：
  - 使用固定类别。
- 列表展示：
  - 显示每个科目的 `错题数量`。

### 右上角

- 功能入口：
  - `举一反三`

### 举一反三

- 打开方式：
  - 点击 `右上角` 的 `举一反三`，进入 `举一反三功能`。
- 页面展示：
  - `主页面` 放题目。
- 答案展示：
  - 页面下方一开始遮挡 `答案`。
  - 由用户主动选择是否展示 `答案`。
  - 展示 `答案` 之后，主动询问用户是否开启 `思考模式` 进行讲解。

## 【后端】产品页面功能描述及交互描述

### 说明

- `后端` 包含 `用户上传图片` 与 `用户进入错题本` 两条流程。
- `用户上传图片` 流程包含 `智能体接收图片`、`智能体分析题目` 与 `智能体主动询问用户是否将本题载入错题本`。
- `用户进入错题本` 流程包含 `分析界面` 与 `举一反三界面`。

### 用户上传图片

- 触发方式：
  - 用户上传图片。
- 后续流程：
  - `智能体接收图片`
  - `智能体分析题目`
  - `智能体主动询问用户是否将本题载入错题本`

### 智能体接收图片

- 处理行为：
  - 进行 `OCR扫描`。
  - 提取文字。

### 智能体分析题目

- 解析行为：
  - 首先对题目进行完全解析，清楚每一步解题步骤。
- 模式询问：
  - 询问用户是否想进入 `思考模式`。
  - 此处需要推荐 `思考模式`。

### 拒绝进入思考模式

- 触发条件：
  - 用户拒绝进入 `思考模式`。
- 系统行为：
  - 直接生成答案。
- 后续入口：
  - 界面下方有开关，用户可随时进入 `思考模式`。

### 同意后进入思考模式

- 触发条件：
  - 用户同意进入 `思考模式`。
- 系统行为：
  - 主动询问用户是哪里不会。
  - 根据用户回答，结合解题步骤回答用户问题。

### 思考模式回答逻辑

- 回答顺序：
  - 先回答用到的相关知识点。
  - 再回答知识点如何在这道题目里适用。
- 图片辅助：
  - 根据问题判断是否需要生成图片帮助理解。
  - 如果需要，主动询问用户是否需要生成图片帮助理解。
- 理解确认：
  - 回答完问题之后，询问用户这道题能否独立解答出来。

### 用户可以独立解答

- 触发条件：
  - 用户回答可以解答出来。
- 系统行为：
  - 对用户进行鼓励。

### 用户无法独立解答

- 触发条件：
  - 用户没办法回答出来。
- 系统行为：
  - 根据解题步骤询问用户是不是下一个知识点没懂或者不会应用。
- 循环规则：
  - 进入循环，直到用户说完全理解题目。

### 智能体主动询问用户是否将本题载入错题本

- 触发方式：
  - 完成题目讲解或答案生成后，智能体主动询问用户是否将本题载入 `错题本`。
- 用户拒绝：
  - 拒绝则不记录本题目与本次对话。
- 用户同意：
  - 同意则将本题目记录进 `错题本`。
  - 总结此次对话。
  - 生成用户的 `知识薄弱点`。
  - 将 `知识薄弱点` 记录到 `错题本` 里的 `分析功能`。
  - 与之前的记录综合成新的文档。

### 用户进入错题本

- 分析界面：
  - 用户可以查看 `分析界面`，理解自己被总结的 `薄弱知识点`。
- 举一反三界面：
  - 用户可以进入 `举一反三界面`。
  - 用户可以练习智能体根据 `错题本` 与 `薄弱知识点` 生成的对应题目。
  - 在出题的同时，系统在旁边写明这道题目考察的是什么知识点。
