# 基于 `langchain-ai/react-agent` 二开 AI 搜题智能体

## 一、任务背景

请基于 GitHub 仓库 `https://github.com/langchain-ai/react-agent.git` 进行二次开发，构建一个用于小学数学搜题与启发式讲解的后端智能体系统。

本项目不是普通“拍照搜答案”工具，而是一个 **AI 搜题陪练 Agent**。核心目标是：

> 接收学生上传的题目图片或手动输入的题目文本，识别题目、解析题意、生成标准解法，并优先引导学生进入“思考模式”；在思考模式中，Agent 不从头到尾直接讲完，而是围绕学生提出的问题和卡点进行针对性讲解、必要时生成图像式解释，并在学生理解后引导其尝试作答；最后支持错题本保存、对话摘要沉淀、薄弱点分析和举一反三练习生成。

本次开发目标是 **后端 Agent + 最小测试接口**，不需要实现完整前端页面。

## 四、智能体业务逻辑

### 4.1 搜题智能体主流程

整体流程：

```text
接收图片或文本
↓
如果是图片：调用多模态 LLM OCR 工具解析题目文字
如果是文本：直接进入题目解析
↓
解析题目
↓
生成标准答案和分步解题步骤
↓
询问用户是否进入思考模式，并推荐思考模式
↓
如果用户否定：直接展示简版答案，并保留进入思考模式的入口
↓
如果用户肯定：询问用户哪里不明白
↓
根据学生回答诊断卡点
↓
先讲解对应知识点
↓
说明知识点如何应用到当前题目
↓
判断是否需要图像式解释
↓
如果需要：生成可视化 JSON
↓
询问学生是否可以自己试着解出来
↓
如果学生还没懂：继续循环卡点诊断与讲解
↓
如果学生已经懂了：让学生尝试作答
↓
校验学生答案
↓
展示标准答案并鼓励用户
↓
询问是否加入错题本
↓
如果用户同意：保存题目、对话摘要、薄弱点分析
↓
如果用户拒绝：不加入正式错题本，但可保留当前线程对话状态
```

### 4.2 错题本智能体流程

```text
用户进入错题本
↓
系统读取默认测试用户的错题记录
↓
按照固定科目分类展示，不允许模型自创科目
↓
整合每次对话摘要，生成结构化错题分析
↓
根据错题库和薄弱知识点生成 3 道举一反三题
↓
每道题包含：题目、答案、解析、考察知识点、难度、来源错题 ID
```

### 4.3 固定科目分类

只能使用以下科目：

```python
ALLOWED_SUBJECTS = ["数学", "语文", "英语"]
```

MVP 阶段：

- 数学：完整支持。
- 语文、英语：只保留分类占位，不实现深度讲解。
- 如果题目不是小学数学，返回：

```text
当前 MVP 主要支持小学数学题目。这个题目我可以先帮你做基础识别，但深度讲解能力暂不完整。
```

TOOLS = [

### 8.2 extract_problem_text_from_image

功能：

- 接收 `image_url` 或 `image_base64`。
- 使用多模态 LLM 识别题目文字。
- 只返回题目相关文字，不要输出无关图片描述。
- 如果无法识别，返回 `success=False`。

输出：`OCRResult`。

注意：

- 发送给多模态模型的 content item 只能包含 `text` 和 `image_url`。
- base64 图片需要转为 `data:image/png;base64,...` 或 `data:image/jpeg;base64,...`。
- 如果当前配置模型不支持多模态，返回错误提示，不要崩溃。

### 8.3 parse_problem

功能：

- 解析题目文本。
- 判断科目、年级、题型、条件、问题、知识点、难度。
- 科目只能是 `数学`、`语文`、`英语`。
- 小学数学题完整解析；其他科目只做基础分类。

输出：`ProblemParseResult`。

### 8.4 generate_solution_plan

功能：

- 对小学数学题生成标准答案。
- 拆分解题步骤。
- 每一步包含：步骤标题、步骤内容、相关知识点、常见误区。
- 同时进行答案自检。

输出：`SolutionPlan`。

### 8.5 diagnose_stuck_point

功能：

- 根据学生输入的“不明白的地方”、当前题目、解题步骤和历史对话判断卡点。
- 卡点可能包括：
  - 题意理解
  - 数量关系
  - 运算选择
  - 公式理解
  - 计算过程
  - 单位换算
  - 知识迁移
- 判断是否需要图像式解释。

输出：`StuckPointResult`。

### 8.6 generate_targeted_explanation

功能：

- 不要从头到尾讲完整题目。
- 只围绕学生当前卡点讲解。
- 讲解顺序固定：
  1. 先讲对应知识点。
  2. 再说明该知识点如何用在当前题目里。
  3. 最后给一个启发式问题。
- 语言要适合小学生。
- 不要一上来直接给最终答案。

输出纯文本。

### 8.7 generate_visual_spec

功能：

- 不调用图像生成模型。
- 生成结构化可视化 JSON。
- 前端未来可用 SVG 或 Canvas 渲染。

支持以下 `visual_type`：

- `bar_model`：线段图 / 条形图
- `number_line`：数轴图
- `blocks`：方块图
- `array`：阵列图
- `fraction_bar`：分数条
- `geometry_sketch`：简单几何示意图

输出示例：

```json
{
  "visual_type": "bar_model",
  "title": "数量关系线段图",
  "description": "用两条线段表示小明和小红的数量关系。",
  "data": {
    "entities": [
      {"name": "小明", "value": 12},
      {"name": "小红", "relation": "more_than", "base": "小明", "diff": 5}
    ]
  },
  "hint": "小红是在小明的基础上多了 5 个，你觉得应该用加法还是减法？"
}
```

### 8.8 check_student_answer

功能：

- 学生说“懂了”后，不要马上展示标准答案。
- 先让学生尝试作答。
- 接收学生答案，与标准答案比较。
- 返回正确性、反馈、标准答案和错误原因。

输出：`AnswerCheckResult`。

### 8.9 summarize_conversation

功能：

- 对本题对话进行总结。
- 提炼学生薄弱点、错误类型、推荐复习内容。
- 用于错题本分析。

输出：`ConversationSummary`。

### 8.10 save_to_wrongbook

功能：

- 将题目加入错题本。
- 保存题目、答案、解题步骤、对话摘要、薄弱点。
- 更新学生画像。

输出：保存结果 JSON。

### 8.11 generate_practice_items

功能：

- 根据错题本和学生薄弱点生成 3 道举一反三练习题。
- 每道题包含：
  - 题目
  - 答案
  - 解析
  - 考察知识点
  - 难度
- 题目应与原错题考点相近，但不能完全复制原题。

输出：`list[PracticeItemSchema]`，数量固定为 3。

## 九、Prompt 设计

请改造 `src/react_agent/prompts.py`。

### 9.1 系统 Prompt 核心原则

系统角色：

```text
你是一个面向小学生的 AI 搜题陪练 Agent。你的目标不是让学生直接抄答案，而是通过启发式提问、针对性讲解、图像式解释和举一反三练习，帮助学生真正理解题目。
```

必须遵守：

- 默认推荐学生进入思考模式。
- 不要直接从头到尾讲完整题目，除非用户明确拒绝思考模式。
- 思考模式下，先问学生哪里不明白。
- 根据学生问题判断卡点。
- 讲解时先讲知识点，再讲知识点如何用于当前题目。
- 必要时生成图像式解释 JSON。
- 用户说懂了后，先让用户尝试作答，再展示标准答案。
- 回答要适合小学生理解，短句、清楚、有鼓励。
- 不输出隐藏思考链。
- 不编造不确定内容。
- OCR 不清楚时要求用户重拍或手动修改题目。
- 数学题低置信度时必须提示可能不准确。

### 9.2 关键话术

推荐思考模式：

```text
这道题我可以直接给你答案，也可以先带你一步步想出来。推荐你试试思考模式，这样你下次遇到类似题会更容易做出来。要进入思考模式吗？
```

询问卡点：

```text
你觉得这道题哪里最不明白？是题意、数量关系、该用什么方法，还是计算过程？
```

启发式连接：

```text
你可以先试着继续做下一步，或者，你希望我继续讲下一个知识点？
```

学生懂了后：

```text
太好了！那你先试着写出自己的答案，我帮你检查一下。
```

保存错题：

```text
这道题里有一个值得复习的卡点，要不要把它加入错题本？我可以帮你记录这次卡住的地方，之后给你出类似题练习。
```

- 如果输入包含 `image_url` 或 `image_base64`，先进入 `ocr_if_needed`。
- 如果输入包含 `problem_text`，跳过 OCR。
- 题目解析置信度低于 `0.6`，进入错误提示节点。
- 用户选择 `否`、`不进入思考模式`、`直接给答案`，进入 `direct_answer_node`。
- 用户选择 `是`、`进入思考模式`、`带我想`，进入 `thinking_mode_node`。
- 思考模式中，若用户表示“还没懂”，继续 `diagnose_stuck_point_node` 循环。
- 用户表示“懂了”，进入 `ask_try_answer_node`。
- 学生提交答案后进入 `check_answer_node`。
- 用户同意保存错题，进入 `save_wrongbook_node`。
- 用户拒绝保存，结束当前流程。

### 10.3 兼容 ReAct 工具调用

可以保留 ToolNode，但不要使用搜索工具。

- 工具调用只用于 OCR、题目解析、解题规划、卡点诊断、图像 JSON、错题保存、举一反三。
- ToolNode 执行后必须回到对应业务节点或模型节点。
- 不要让 Agent 无限循环调用工具。
- 设置合理 recursion limit。
