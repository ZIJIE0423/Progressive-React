# 需求文档

## 引言
我们是一款AI教育产品，核心场景为“扫码搜题”。当前存在两个核心问题：  
- 学生端：82%的学生在搜题后直接点击“查看全解”，平均停留时间仅12.35秒，后续“举一反三”练习中正确率仅为35%。  
- 家长端：家长反馈“孩子依赖搜题，不动脑筋”，学生认为“AI提示太生硬，看不懂”。  

业务目标：  
- 将“引导采纳率”（参与互动的用户占比）从10%提升至30%  
- “举一反三”练习正确率从35%提升至50%+  

本EARS需求文档针对上述问题，提出三个核心功能需求，旨在通过针对性提问引导、可视化讲解及卡点沉淀机制，提升学生思考参与度与知识迁移能力。

## 需求

### 需求 1
**用户故事:** 作为学生，我想要与系统有互动感，系统识别我当前的卡住点并进行分步追问，以便自己完成关键推理过程。

**验收标准:**  
1. 当学生提交问题查询时，系统应分析当前步骤并识别潜在卡点  
2. 如果学生的查询缺乏足够上下文，系统应在继续引导前提示澄清  
3. 当系统识别到卡点时，应生成2-3个针对性问题引导学生完成推理  
4. 如果学生在提问过程中提供错误答案，系统应提供提示而非直接给出答案  
5. 当学生成功完成推理后，系统应确认正确答案并提供简要总结

### 需求 2
**用户故事:** 作为小学生，我想要系统根据题型自动匹配可视化图表，以便更直观理解抽象的数量关系。

**验收标准:**  
1. 当系统检测到小学数学题型（如加减法应用题、倍数关系等），应自动匹配并生成相应的可视化模板（线段图、数轴图等）  
2. 如果OCR识别准确率低于85%（手写体或公式识别），系统应显示基础可视化模板并附带"输入质量提示"  
3. 当题目同时符合多种可视化模板时，系统应优先选择与题型最匹配的模板  
4. MVP阶段系统应支持以下高频题型：加减法应用题、倍数关系、乘除法分组、分数比较  
5. 每次生成可视化图表后，系统应结合图表提供1-2个启发式问题引导学生思考

### 需求 3
**用户故事:** 作为学生，我想要系统记录我的常错知识点并进行总结，生成个性化练习题，以便巩固薄弱环节。

**验收标准:**  
1. 当学生在解题过程中出现错误时，系统应记录错误类型、题目类型及问答上下文信息，动态掌握学生卡点  
2. 如果学生对同一知识点连续出错2次以上，系统应标记该知识点为"需强化"并优先生成迁移练习  
3. 在完成卡点解释后，系统应自动生成1道参数变化但题型相同的迁移练习题  
4. 迁移练习题的难度应基于学生历史表现动态调整，浮动范围不超过±20%  
5. 当学生正确完成迁移练习题时，系统应更新该知识点的掌握状态为"已巩固"
